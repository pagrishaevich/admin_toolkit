## Исправление CIFS-доступа

Что изменилось:

- Разделены CIFS mount options для `/mnt/inv` и `/mnt/distr`.
- `/mnt/inv` теперь получает `rw,noperm,file_mode=0777,dir_mode=0777`.
- `/mnt/distr` теперь получает `ro`.
- `scripts/cifs.sh` очищает старые дублирующиеся строки `/mnt/inv` и `/mnt/distr` в `/etc/fstab` и оставляет по одной актуальной записи.
- `scripts/cifs.sh` перемонтирует шары, чтобы новые права применялись сразу.
- `postcheck` проверяет возможность записи в `/mnt/inv`.
- README и `install.md` обновлены.

Проверка:

- На `10.82.100.36` выполнены `validate.sh` и реальный `bootstrap.sh --step cifs`.
- `/mnt/inv` смонтирован `rw` с `file_mode=0777`, `dir_mode=0777`, `noperm`; запись от пользователя `nobody` прошла.
- `/mnt/distr` смонтирован `ro`; запись от пользователя `nobody` отклонена с ошибкой read-only filesystem.

## Скрипты миграции перед переустановкой РедОС

Что изменилось:

- Добавлена спецификация `docs/superpowers/specs/2026-05-15-redos-reinstall-migration-design.md`.
- Добавлен план `docs/superpowers/plans/2026-05-15-redos-reinstall-migration-implementation.md`.
- Добавлен `backup_restore/scripts/preinstall_backup.sh` для сбора backup перед установкой РедОС 8.
- Добавлен `backup_restore/scripts/postinstall_restore.sh` для восстановления backup в профиль доменного пользователя.
- Добавлен `backup_restore/tests/migration_test.sh`, а корневой `scripts/validate.sh` запускает проверку подпроекта.
- README получил раздел "Миграция перед переустановкой РедОС".

Состав backup:

- сетевые папки из `fstab`, активных mount, `/media`, `/run/media`, `/mnt` и GVFS;
- профиль Яндекс Браузера без кэша;
- ViPNet отдельно от Ассистента;
- Ассистент как отдельное ПО для удаленного подключения;
- CUPS-принтеры и SANE/USB-сканеры;
- `/etc/pki` и пользовательские NSS-хранилища;
- КриптоПро, Rutoken и Jacarta из типовых конфигов и диагностических команд;
- `restore-hints.txt` с проверками после установки.

Что намеренно не сохраняется:

- полный список RPM-пакетов;
- `/etc/NetworkManager/system-connections`;
- доменное состояние;
- SSH-ключи пользователя.

Проверка:

- На `10.82.100.36` через PuTTY/plink выполнены `bash scripts/validate.sh` и help-проверки из временной копии проекта в `/tmp/admin_toolkit_migration_test`.
- На `10.82.100.36` выполнен реальный backup для пользователя `nagrady` в `/home/migration-test-backup`; создан каталог `redos-migration-GUB-ADM-TEST-20260515-172925` размером около 27 МБ.
- В backup проверены `manifest.txt`, `restore-hints.txt`, `mounts`, `certificates`, `cryptopro`; ожидаемо не найдены профиль Яндекс Браузера, ViPNet и Ассистент.
- После backup SSH начал сбрасывать соединение (`Connection reset by peer`), поэтому автоматическая очистка тестовых каталогов не подтверждена. При следующем доступе удалить:
